## 1. Design intent (≤ 60 words)

- **Named direction:** **Event Horizon** — cinematic, dark-first deep-space immersion: full-bleed cosmic gradients, a drifting parallax starfield behind the hero, luminous accent glow, oversized display type, generous letterboxed spacing. Motion that feels like coasting through orbit.
- **Why this direction:** A premium spaceflight brand sells *awe*; the site should feel like the brand's flagship film at night.
- **Divergence from other variants this run:** v2 ("Daybreak Atlas" — light, editorial, grid-driven, restrained) is the calm counterpoint; **Event Horizon** is its opposite — dark, atmospheric, dramatic.

## 2. Inherited baselines (do not redescribe — link)

This variant **inherits and does not restate**:
- **Theme contract** — light / dark / system, three-state, persisted, FOUC-safe, hydration-safe per **ADR-0004** + `custom_content-sdk-theme-hydration-pattern`. Tokens live in the Tailwind v4 `@theme` block (`globals.css`), `@custom-variant dark`. Hex-literal tokens are used via `var(--token)` directly — **never** wrapped in `hsl()`.
- **Accessibility floor** — WCAG 2.1 AA contrast on **both** themes; semantic landmarks; visible focus; `prefers-reduced-motion` honored (ADR-0004, PRD FR-6 / NFR-3 / NFR-6).
- **Component prop boundary** — leaf = flat props, container = folder-of-children, nav excluded (**ADR-0005**). This spec invents no content shape a single field can't carry.
- **Forms** — Newsletter + Contact are presentational-only with default/focus/error/success/empty states (**ADR-0006**).
- **NOT a Marketplace app** — **Blok is not used.** Event Horizon defines its own bespoke cosmos token set below; ignore all Blok enforcement.

## 3. Direction deltas (what Event Horizon establishes — bespoke cosmos design system)

> The POC is the visual source of truth. Below = token/system definitions the Lead Developer wires; visual composition is shown in the POC, not re-prosed here.

### 3a. Type system
- **What:** Display = **Fraunces** (optical serif, high-contrast, set tight at large sizes for cinematic headlines). Body/UI = **Geist Sans** (or Inter as fallback) for clean legibility. Numerals/stat values = Geist tabular.
- **Why:** Serif display carries luxury + editorial gravitas; clean sans keeps body and UI calm under the drama (the personal-site Cormorant+Jost lesson, Aphelion's own pairing).
- **Where in POC:** hero headline, all `PageHero`, section eyebrows. `screen-1.html § hero`.
- **Hook:** fluid `clamp()` scale on `--step-*` custom props; display weight 300–400 only (light/regular optical), never bold serif.

### 3b. Color tokens — BOTH themes (hex literals, used as `var(--token)` directly)

**Dark theme (`:root`, the brand "night sky"):**

- **Why two glow bases:** the hero radial-gradient + starfield is the signature device; each theme needs its own glow base so the device reads as "night sky" (dark) vs "dawn cosmos" (light), not a recolored dark screen.
- **Hook:** accents via `*-500` tokens; alpha glows via `color-mix(in oklch, var(--accent-glow) 40%, transparent)` or SVG `stopOpacity` — **never** `hsl(var(...))`.

### 3c. Spacing & layout rhythm
- **What:** Generous cinematic spacing — section vertical rhythm `clamp(5rem, 12vh, 9rem)`; max content width `~1200px` centered with wide letterboxed gutters at 1440. Full-bleed bands (hero, promo, stats) break the container to edge-to-edge.
- **Why:** Letterboxing + breathing room reads as premium/filmic; content bands alternate full-bleed cosmic vs contained.
- **Where in POC:** `screen-1.html` band rhythm; gutters visible at 1440.

### 3d. Motion language ("drift")
- **What:** (1) hero parallax starfield — slow multi-layer drift (canvas/SVG, layers move at different rates on scroll); (2) fade+rise on scroll-reveal for bands; (3) hover elevation + accent-glow bloom on cards; (4) gentle CTA glow pulse.
- **Why:** Evokes coasting through orbit — "tasteful, not overloaded" (PRD NFR-3). No autoplay media, nothing fighting readability.
- **Reduced-motion:** all of the above collapse to static (starfield → static star SVG, reveals → instant, pulse → off) under `prefers-reduced-motion: reduce`.
- **Hook:** single `prefers-reduced-motion` guard at the CSS root + a JS `matchMedia` check (in `useEffect`, never render-body) before starting the starfield rAF loop.

### 3e. Signature visual devices
- Radial hero glow + drifting parallax starfield (the hallmark). Hairline orbital-ring SVG decorations behind stats. Luminous accent underlines on links. Glass-edge elevated cards (subtle `--bg-elev` + 1px `--line` + soft outer glow on hover). Monochrome glyph icons only (Lucide-style `✓` / `✕`, never emoji codepoints) for form state.

## 6. Accessibility notes

- **Contrast (to verify on POC via `getComputedStyle`, AA target ≥4.5:1 text / ≥3:1 large+UI):** dark `--fg #eef1fb` on `--bg #05060f` (~17:1, pass); dark `--accent-500 #6d8cff` on `--bg` for large/link (verify ≥3:1, expected pass; underline added so link affordance isn't color-only); light `--fg #10142b` on `--bg #f4f5fb` (~15:1, pass); light `--accent-500 #3a55d9` on `--bg` (verify ≥4.5:1). Final ratios captured on the POC, not claimed here.
- **Keyboard flow:** logical tab order header → hero CTA → bands → footer; theme toggle and mobile drawer fully keyboard-operable; drawer traps focus while open and closes on `Esc`.
- **Reduced-motion:** `prefers-reduced-motion: reduce` → starfield static, scroll-reveals instant, CTA pulse off (POC honors this; verify by toggling).
- **Forms:** error messages tied via `aria-describedby`; invalid fields `aria-invalid`; success uses `role="status"` live region. State icons are monochrome glyphs (`✓`/`✕`) that inherit theme color — **no emoji codepoints**.
- **Imagery:** all decorative cosmic gradients/starfield `aria-hidden`; content images carry alt text.
