## 1. Design intent (≤ 60 words)

- **Named direction:** **"Pulse" — Nebula Drift with PORTAL WINDOWS, depth & cursor-reactive light.** A *variation of operator-chosen v5*: keeps the drifting neon mesh + kinetic gradient DNA, then adds real cosmos photos masked into glowing neon-ring portals, 3D cards that tilt toward the cursor, a pointer-tracking hero glow, a scroll-progress aurora, and a breathing focal orb. Alive, deep, tactile.

## 3. Direction deltas (what "Pulse" adds over v5 — see POC for visuals)

> POC is the visual source of truth. Below = the *new* devices the Lead Developer wires; composition lives in `pocs/poc-v5c-prd000/index.html` + `styles.css`.

- **3a. Portal windows (signature).** Real royalty-free cosmos photos masked into circular (`.portal`) or top-arched (`.portal.arch`) windows. Each has an animated **conic neon ring** (`::before`, masked border, `--ring-grad`, spins 14s) and a **pulsing bloom halo** (`::after`, 4.5s). Used in: hero cluster (Earth-limb + astronaut), 6 destination cards, 3 experience items, promo. *See `index.html § hero`, `§ destinations`, `§ experiences`, `§ promo`.*
- **3b. Image sourcing + offline fallback.** Photos hotlinked from **Unsplash (Unsplash License, royalty-free)** at `images.unsplash.com/photo-<id>?w=1200&q=80`; `loading="lazy"` (hero eager) + descriptive `alt`. **Every `<img>` has a CSS mesh-gradient fallback** layer (`.portal-fallback`, `--img-fallback`) behind it; `theme.js` hides a failed `<img>` so the mesh shows through. PRD-000 §6 imagery compliance: royalty-free only, no branded assets. Full URL list in `index.html` `<!-- IMAGE SOURCES -->`.
- **3c. 3D tilt cards.** `.tilt` shells set `transform-style: preserve-3d`; `theme.js` reads pointer position and applies `rotateX/Y` (±9°); inner `.lift` layers (`translateZ`) give parallax depth (portal floats forward, badge floats further). Skipped on coarse pointers and under reduced-motion. *Hero cluster, destination cards, promo portal.*
- **3d. Cursor-reactive hero glow.** `.cursor-glow` radial-gradient follows the pointer via `--mx/--my` CSS vars (`mix-blend: screen`), strength per-theme (`--cursor-glow-strength`). Fades in on hero hover only.
- **3e. Scroll-progress aurora.** Fixed 3px neon bar (`.scroll-aurora`) pinned top; `scaleX(var(--scroll))` driven by a rAF-throttled scroll listener; glows via `box-shadow`.
- **3f. Pulse / breathing.** Hero orb now **breathes** (`orbBreathe` 6s — scale + shifting bloom color) instead of v5's gentle drift; portal rings pulse (`ringPulse`); brand mark slow-spins.
- **3g. Hero recomposition.** v5's single-column hero becomes a **2-col split** (copy + portal cluster) at desktop, stacking at ≤980px. Kinetic gradient now also drives the hero stat numerals.

## 6. Accessibility notes (verified on POC via getComputedStyle)

- **Contrast — AA pass BOTH themes.** **Dark:** fg/bg **17.82:1**, muted/bg **9.03:1**, dest-link/surface **5.29:1**. **Light:** fg/bg **16.24:1**, muted/bg **6.92:1**, dest-link/white **6.00:1**, tag/white **7.56:1**. All ≥4.5:1 text / ≥3:1 UI. Body text never rides neon/photos — only solid surfaces. Links carry an underline/arrow affordance, not color-only.
- **Gradient-clipped headlines** are large display (≥30px) with a solid `--fg` fallback before `background-clip` (no-clip engines still read them).
- **Imagery:** content photos carry descriptive `alt`; mesh, orb, rings, stars, cursor-glow, scroll-aurora all `aria-hidden`.
- **Keyboard:** logical order header→hero→bands→footer; visible focus ring on all interactives; mobile drawer **focus-trapped, Esc-closes, returns focus to trigger** (verified). Newsletter errors via `aria-invalid` + `aria-describedby`; success `role="status"`. State glyphs are monochrome `✓`/`✕` (no emoji codepoints).
- **Reduced motion (verified in CSS + JS):** `prefers-reduced-motion: reduce` → mesh frozen, kinetic type static, **orb pulse off, 3D tilt disabled, cursor-glow hidden, scroll-aurora flat**, reveals shown. JS also bails its rAF/tilt/glow paths on `reduceMotion.matches` and on a runtime OS toggle.
