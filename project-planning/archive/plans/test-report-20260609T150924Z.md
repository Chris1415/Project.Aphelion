## 6. Fixes Applied

### m-4 — ContactForm: contradictory `role="alert" aria-hidden="true"` on always-present empty placeholder elements

**File:** `static/src/components/ContactForm/index.tsx`
**Change:** Removed all three `{!fieldError && <p className="field-error sr-only" id="cf-*-error" role="alert" aria-hidden="true" />}` branches (name field ~line 159, email field ~line 185, message field ~line 210). Each field now only renders `{fieldError && <p id="cf-*-error" role="alert">...</p>}` when an error is present. The `aria-describedby` attributes on the inputs remain; pointing to a non-existent ID is harmless per the ARIA spec.

**Rationale:** An element cannot simultaneously declare a live region (`role="alert"`) and be hidden from the accessibility tree (`aria-hidden="true"`). Some AT implementations behave unexpectedly with this pattern.

**Post-fix verification:**
- `npm run lint`: 0 errors — PASS
- `npx tsc --noEmit`: 0 errors — PASS
- `npm run test` (Vitest): 100/100 — all 9 ContactForm tests pass, including `role="alert"` assertion (test 9) and aria-describedby wiring (test 8)
- `npm run test:e2e` (Playwright a11y.spec.ts): 0 serious/critical axe violations on /contact in both themes — PASS
