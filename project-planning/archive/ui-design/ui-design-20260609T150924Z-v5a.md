## 1. Design intent (≤ 60 words)

- **Named direction:** **"Lumen" — Nebula Drift, made photographic.** A v5 variation: real deep-space photography is the hero (full-bleed nebula with neon mesh blended on top + scroll parallax); section imagery is duotone-tinted into the magenta/cyan/violet palette via blend-modes; destination cards reveal real planetary photos behind clip-path masks. Same kinetic gradient headlines, richer motion.
- **Why this direction:** Real cosmos imagery sells *awe* a pure-CSS mesh can only suggest — Lumen keeps Nebula Drift's luminous brand but grounds it in photographable wonder.
- **Divergence from base v5:** v5 is 100% CSS-generated atmosphere; **Lumen adds real photography** (hero, duotone bands, reveal-mask cards) and layered scroll-driven motion (parallax, ken-burns, shimmer) on top of v5's identical token system.

## 2. Inherited baselines (do not redescribe — link)

- **Everything in `ui-design-20260609T150924Z-v5.md` §2 + §3** is inherited verbatim: theme contract (light/dark/system, FOUC-safe, hydration-safe per ADR-0004), a11y floor (WCAG 2.1 AA both themes), component prop boundary (ADR-0005), presentational forms (ADR-0006), bespoke neon token set (NOT Blok), hex via `var(--token)` never `hsl(var())`.
- **Token system inherited from `pocs/poc-v5-prd000/styles.css`** — same `--bg`/`--surface`/`--neon-*`/`--accent-500`/`--text-grad` for BOTH themes, same type scale, same kinetic/mesh/orb utilities. Lumen ADDS photographic-treatment tokens (see §3); it does not change the v5 palette.

## 3. Direction deltas (what Lumen ADDS over base v5)

> POC is the visual source of truth. Below = what's NEW vs v5; everything else = v5 baseline.

### 3a. Photographic full-bleed hero (new)
- **What:** A real full-bleed nebula photo fills the hero; v5's neon `.mesh` rides on top via `mix-blend-mode: screen/overlay` so the photo stays on-brand; a `linear-gradient` **scrim** sits between photo and text for AA. Image drifts on **scroll parallax** (`translateY` driven by scroll position) + slow ken-burns scale.
- **Why:** Awe-first; the photo is the emotional payload, the mesh is the brand veil.
- **Where in POC:** `index.html § hero` (`.hero-photo` + `.mesh` overlay + `.hero-scrim`).
- **Hook:** `--scrim: linear-gradient(...)` per theme; parallax via tiny rAF scroll handler gated by `matchMedia('(prefers-reduced-motion: reduce)')` in a `useEffect`-equivalent inline script — never render-body.

### 3b. Duotone-blended section imagery (new)
- **What:** Real space photos in the Experience showcase + Promo are tinted into-brand: grayscale base + a magenta→cyan→violet gradient layer set to `mix-blend-mode: color/screen`, plus `filter: saturate()/contrast()`. Reads as on-palette duotone, not a raw stock photo.
- **Why:** Keeps real imagery without breaking the neon identity.
- **Where in POC:** `index.html § experiences`, `§ promo` (`.duotone` wrapper).
- **Hook:** `.duotone img{filter:grayscale(1) contrast(1.05)}` + `.duotone::after{background:var(--text-grad);mix-blend-mode:color;opacity:var(--duotone-strength)}`; `--duotone-strength` differs per theme (stronger on dark).

### 3c. Image reveal-mask destination cards (new)
- **What:** Destination cards show real planetary/orbital photos revealed behind a `clip-path` inset mask that animates open on scroll-into-view; hover does a ken-burns push-in on the image.
- **Why:** A premium "unveiling a destination" gesture beyond v5's CSS `.sky` swatch.
- **Where in POC:** `index.html § destinations` (`.dest-media.reveal` + real `<img>`).
- **Hook:** `clip-path: inset(...)` 0→full on `[data-reveal].in`; reduced-motion → mask starts open.

### 3d. Richer motion (extends v5 "drift")
- **What (added to v5's mesh-drift + kinetic type):** (1) hero **scroll parallax** on photo + mesh at different rates; (2) **ken-burns** slow scale-drift on hero + duotone + card images; (3) **gradient-text shimmer** sweep on headlines (faster, brighter than v5's base kinetic); (4) clip-path **reveal masks** on scroll.
- **Reduced-motion:** ALL freeze — parallax off (photo static), ken-burns off, shimmer static, masks start fully open. Single root `@media (prefers-reduced-motion: reduce)` guard + JS `matchMedia` gating the rAF parallax.

### 3e. Imagery fallback (graceful degradation — required)
- **What:** EVERY `<img>` sits over a CSS gradient/mesh fallback background (the v5 `.sky` swatch) so layout reads if an image 404s/offline; `loading="lazy"`, meaningful `alt`. Image URLs + Unsplash source documented in the HTML head comment + §7.
- **Hook:** `.photo-frame{background:var(--sky-swatch)}` behind every `img`; `img` failure leaves the on-brand gradient visible.

## 6. Accessibility notes

- **Text-over-image scrim strategy (Lumen-specific):** No body/heading text sits directly on a photo. The hero photo region is **dark-scrimmed in BOTH themes** (scrim anchors to `#0a0518` over the photo), so hero copy uses a dedicated **`--hero-text` token that stays light in both themes** (NOT `--fg`) — this was a real fix caught on the POC: light-theme `--fg` is dark and failed over the dark scrim. Card/section copy rides solid `--surface`/`--bg`, never the duotone image. Destination labels over photos get a local bottom scrim.
- **Contrast — VERIFIED on POC via `getComputedStyle` + WCAG ratio calc (AA = 4.5:1 text / 3:1 large):** DARK — body fg/surface **16.74:1**, fg/bg **17.82:1**, hero-text/scrim **17.68:1**. LIGHT — body fg/surface **17.74:1**, fg/bg **16.24:1**, hero-text/scrim **18.34:1**. All pass AA with large margin. `--accent-500` links (dark `#c64bff`, light `#8a2be0`) underlined (not color-only). Kinetic gradient headlines are LARGE display only with solid `--fg`/`--hero-text` fallback before `background-clip`.
- **Reduced-motion:** `prefers-reduced-motion: reduce` → hero parallax off (photo static), ken-burns off, shimmer static, reveal masks start fully open, mesh frozen (verify by toggling on POC).
- **Imagery:** decorative mesh/orb/scrim/particles `aria-hidden`; every content `<img>` has meaningful `alt`; gradient fallback ensures no broken-image gap.
- **Keyboard:** header → hero CTA → bands → footer; theme toggle + drawer operable; drawer traps focus, `Esc` closes, returns focus to trigger.
- **Forms:** `aria-describedby` + `aria-invalid`; success `role="status"`; monochrome glyphs (`✓`/`✕`) — no emoji codepoints.
