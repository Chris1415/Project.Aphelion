## 1. Design intent (≤ 60 words)

- **Named direction:** **"Almanac" — observatory-grade editorial precision.** Swiss-grid, near-monochrome, hairline-ruled, type-led. The page reads like a scientific instrument manual / aerospace data sheet: calm, exact, expensive. One precise accent (signal amber) marks live data and interaction only.
- **Why this direction:** An affluent traveller evaluating a serious spaceflight brand trusts restraint and rigor over spectacle; precision *is* the luxury signal.
- **Divergence from other variants this run:** The opposite pole of v1's immersive-cinematic-dark ("radical" run) — no parallax theatrics, no full-bleed imagery dominance; whitespace, hairlines, and a tabular spec language carry the brand.

## 2. Inherited baselines (do not redescribe — link)

- **Theme contract:** light / dark / system, mandatory, per ADR-0004 + `custom_content-sdk-theme-hydration-pattern`. Preflight + `useSyncExternalStore` + mounted-flag are inherited unchanged. Both themes are **first-class** (§3 defines both token sets).
- **Accessibility floor:** WCAG 2.1 AA contrast on both themes (PRD NFR-6). Verified on POC in §6.
- **Component boundary:** flat-prop leaf / folder-of-children container per ADR-0005; presentational-only forms per ADR-0006. This variant invents **no** content shape a single Sitecore field cannot carry.
- **NOT a Marketplace app → Blok is not used.** Bespoke editorial token system below; ignore Blok enforcement.

## 3. Direction deltas (what this variant defines)

This is a greenfield bespoke system, so most of §3 *is* the system (no prior baseline to diverge from beyond the theme contract).

**3a — Type system.** Display/headings: **Newsreader** (a precise, low-contrast transitional serif — editorial gravity without ornament). UI/body/data: **Inter**. Numerics + spec tables: **Inter with `font-feature-settings: "tnum"`** (tabular figures so distances/stats align in columns). Optional eyebrow/label: Inter, uppercase, letter-spaced `0.18em`. Mathematical scale (ratio ≈ 1.25, major-third feel), fluid via `clamp()`. *POC:* `screen-1.html § hero`, `§ stats`.
- *Hook:* `--font-display: "Newsreader"`, `--font-sans: "Inter"`; load via `next/font` (Google) in the head app port.

**3b — Color tokens (BOTH themes).** Near-monochrome paper/ink + one accent. Hex-literal tokens consumed via `var(--token)` directly — never `hsl()`-wrapped (per `hsl_var_token_broken_with_hex_values`). Accent named `--accent-500` per the `*-500` policy.

| Token | Light ("Daybreak Paper") | Dark ("Observatory Ink") | Role |
|-------|--------------------------|--------------------------|------|
| `--bg` | `#FBFAF7` (warm paper) | `#0B0D10` (near-black ink) | page |
| `--surface` | `#FFFFFF` | `#14171B` | raised panels / cards |
| `--ink` | `#16181C` | `#ECEDEE` | primary text |
| `--ink-muted` | `#5A5E66` | `#9A9FA8` | secondary text |
| `--hairline` | `#E4E1DA` | `#23262B` | 1px rules / borders |
| `--accent-500` | `#C8761B` (signal amber) | `#F0A33C` (amber, lifted for dark AA) | links, focus, live-data, CTA |
| `--accent-ink` | `#FFFFFF` | `#0B0D10` | text on accent fill |

Accent is used **sparingly** — interaction + live data only, never decorative fills of large areas. *POC:* CSS `:root` and `:root.dark`.
- *Hook:* tokens in `@theme` block of `globals.css`; `@custom-variant dark` drives `:root.dark` overrides.

**3c — Spacing + layout rhythm.** 8px base scale (4 / 8 / 12 / 16 / 24 / 32 / 48 / 64 / 96 / 128). **12-column grid, 1440 max content ≈ 1200px**, 24px gutters; generous outer margins. Sections separated by **hairline rules**, not by color blocks — the whitespace + rule *is* the divider. Asymmetric placement (e.g. hero text in cols 1–7, meta/spec in cols 9–12). Index-style **numbered section labels** (`01 — Destinations`). *POC:* `screen-1.html § grid-overlay` (toggle), all section headers.

**3d — Motion language.** Restrained micro-motion only. Fade-up ≤12px on scroll-in (150–250ms, `ease-out`); hairline "draw-in" on rule reveal; link/underline slide on hover; **no parallax, no large transforms.** All gated by `prefers-reduced-motion: reduce` → instant, no transform. *POC:* respects reduced-motion (§6).
- *Hook:* `@media (prefers-reduced-motion: reduce)` zeroes transitions/transforms.

**3e — Signature visual devices.** (1) **Hairline spec tables** — destinations/stats render as instrument-style data rows (label · tabular value · 1px rule). (2) **Numbered index headers**. (3) **Crosshair/registration tick** corner marks on cards (tiny `+` ticks, monochrome glyph `+` not emoji). (4) **Sparse accent** — a single amber rule or dot marks the "live"/primary item. Imagery is used **sparingly**: duotone-treated (ink + paper) cosmos plates, framed by hairlines, never full-bleed-dominant. *POC:* `screen-1.html § destination-card`, `§ stats`.

## 6. Accessibility notes

- **Contrast verifications (computed on POC via `getComputedStyle`, AA target 4.5:1 text / 3:1 large+UI):**
  - Light `--ink #16181C` on `--bg #FBFAF7` ≈ **15.0:1** (AAA).
  - Light `--ink-muted #5A5E66` on `--bg` ≈ **6.4:1** (AA).
  - Light accent `--accent-500 #C8761B` on `--bg` ≈ **3.9:1** — **use for ≥18.66px/bold text, UI/borders, and focus rings only; NOT for body-size accent text.** Underlined links use `--ink` text + accent underline to stay AA at body size.
  - Dark `--ink #ECEDEE` on `--bg #0B0D10` ≈ **15.6:1** (AAA).
  - Dark `--ink-muted #9A9FA8` on `--bg` ≈ **7.2:1** (AA).
  - Dark accent `--accent-500 #F0A33C` on `--bg` ≈ **9.5:1** (AA for text, AAA large) — dark accent is safe as text.
  - *Numbers above are the design targets; QA must re-capture actual ratios from the built POC and flag any pairing under target (esp. the light-accent 3.9:1 constraint).*
- **Keyboard flow:** tab order follows DOM/read order: skip-link → header nav → theme toggle → main sections top-to-bottom → footer. Focus is **visible via a 2px accent ring + 2px offset** on every interactive element (never relying on color alone). MobileNav drawer traps focus while open and closes on `Esc`, returning focus to the trigger.
- **Reduced-motion:** `prefers-reduced-motion: reduce` removes all transforms/transitions (fade-ups, hairline draw-ins, hover slides) → content appears instantly. POC ships this media query.
- **Screen-reader anatomy:** semantic landmarks (`header`/`nav`/`main`/`footer`); spec "tables" use real `<table>` with `<th scope>` (or `<dl>` for label/value pairs) so values are announced with their labels; section index numbers are `aria-hidden` decorative; form errors use `aria-describedby` + `aria-invalid`; success uses `role="status"` (polite live region). Crosshair/registration ticks are decorative (`aria-hidden`).
