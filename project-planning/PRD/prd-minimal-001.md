## Problem

PRD-000 delivered a faithful static app + a Sitecore content-model spec, but no running head app and no live data. Act 3 must stand up the real Content SDK head app against a live XM Cloud tenant, port every component to SDK field components driven by live Edge data, wire navigation, and deploy — without re-litigating the locked v5b design.

## Goal

A deployed Sitecore Content SDK head app (App Router) for the 5 Aphelion routes, rendering **live published content** from the tenant via stock `client.getPage()`, visually faithful to the v5b static app, with real-tenant smoke green. Home route ported + smoke-verified FIRST as a gate before the inner routes.

## Non-negotiables

- **Scaffold via the canonical CLI** (`create-content-sdk-app`, App Router) on top of the baseline copy (`C:/Projects/agentic/SitecoreAI_Authoring`, token-replaced `cosmos`/`aphelion`), per rule `50-scaffold`. NOT `create-next-app` (that was PRD-000). Never hand-write `package.json`/`next.config.*`.
- **100% stock data layer** — the head app reaches Edge via `client.getPage()` etc.; **no mock infrastructure** (the tenant is real).
- **Port = the work.** Each component: plain props → SDK field components (`<Text>` `TextField`, `<RichText>` `RichTextField`, `<Image>` `ImageField`, `<Link>` `LinkField`); field types cited from `node_modules/@sitecore-content-sdk/react` `.d.ts` (rule `40-sdk-contracts`).
- **Derive props from the LIVE `getPage` payload** (probe `/api/layout-debug` per component), NOT from the static app's prop names (the static names guided the content model; the live payload is the contract — this is the #1 port-drift trap).
- **`componentName` registered EXACTLY** in the component-map (mismatch → `<MissingComponent>`). Containers use the folder-of-children (Children resolver) shape; helpers/types/queries live OUTSIDE `src/components/<X>/` (generate-map scanner trap).
- **Authoritative `componentName` registry (single source of truth).** Finalizing `sitecore-content-model.md` MUST produce one canonical table of every `componentName`. The operator's renderings, the head-app component-map, and the live `getPage` payload all bind to that table — all three must match it. Do not restate the component list ad hoc.
- **Content oracle rule (static-vs-live):** the tenant is seeded with the static app's content **verbatim** (the runbook already copies field values from `static/src/content/*`), so the static app is BOTH visual and content oracle. The automated screenshot-diff checks **layout/structure (text-tolerant)**; the **operator eyeball is the fidelity arbiter**. When diff flags text the operator judges.
- **Tenant is built ONCE, fully, up front** (operator, before scaffold boot — build-flow Phase 5.5 single gate): all 5 routes' renderings + datasources + content published to Edge. The **Home-first gate is a head-app PORTING tranche** (port + smoke Home's components first), NOT a staged tenant build.
- **Marquee derivation:** all published **Destination** items, in **Sitecore source order**, the **Name** field only, no truncation. No marquee datasource.
- **Nav** (header/footer — excluded from the content model, ADR-0005) via `custom_content-sdk-component-level-navigation`: server L1 + lazy L2 per-hover via a **Route Handler at `/api/nav-children`** (NEVER a Server Action), no-gap dropdown, intent-delay. **This nav Route Handler SURVIVES to production** — only `/api/layout-debug` is removed at ship.
- **Editing-safe rendering** (App Router: `draftMode()` + `getPreview`/`getDesignLibraryData` from `searchParams`).
- **Theme/fonts port verbatim** from PRD-000 (Clash Display self-hosted, gooey `#gooey` perf guard, the MutationObserver reveal controller, dark+light+system).
- **Real-tenant smoke (split model):** **Automated (Playwright):** 0 `<MissingComponent>`, 0 empty renderings, no full-document reload on nav hover, structural screenshot-diff vs static, axe, motion-ON + client-nav (framework `/test § 1d-ii.5b`). **Operator-manual:** visual-fidelity sign-off (the arbiter), the Pages-editor experience, nav "feel" (no flicker). BOTH required per route.

## In scope / out of scope

- **In scope:** baseline-copy + token-replace + `create-content-sdk-app` scaffold; `.env.local` → real Edge contextId; finalize `sitecore-content-model.md` (3 locked decisions); per-component port of all components across the 5 routes; component-level navigation; editing-safe rendering; real-tenant smoke; Vercel deploy. **Home route is the first hard verify-gate tranche.**
- **Out of scope:** redesign / new UI; i18n / multilingual; personalization / A-B; real forms backend; search; content authoring workflow. (Forms remain presentational, now reading their copy from Sitecore fields.)

## Success criteria

- Head app scaffolded (stock Content SDK, App Router), `.env.local` → real contextId; `/api/layout-debug?site=aphelion&locale=en&path=/` returns real layout.
- All 5 routes render live published content with **zero `<MissingComponent>`** and zero empty/placeholder renderings.
- Each component is SDK-field-driven from the live payload; container components resolve folder-of-children; nav works (hover dropdown, no flicker, no page-reload symptom).
- Visual parity with the v5b static app (screenshot-diff per route, both themes) — the static app is the oracle.
- Editing/preview safe (renders in Pages editor without crashing).
- **Home gate exit criteria (binary — ALL must pass before inner routes start):** automated — 0 `<MissingComponent>` on Home, 0 empty renderings, component-map resolves every Home `componentName`, `build`+`lint`+`tsc` green, nav functional (hover dropdown, no full reload), axe clean, motion-ON + client-nav checks pass, structural screenshot-diff vs static Home passes both themes; **plus operator sign-off** on Home visual fidelity.
- **Real-tenant smoke green** per route (automated + operator-manual split); inner routes only begin after the Home gate passes.
- `npm run build` + `lint` + `tsc` green; deployed to Vercel; final smoke on the prod domain.
